## 2. 逐点精讲

### 知识点 1：加密算法的概念与分类

#### 实际意义

通过一系列操作将信息加密防止他人轻易获得秘密，在各个领域都有其特殊作用；在信息安全方面，加密可以防止外部控制 MCU 来影响系统的操作或者用户数据的泄露以及技术的抄袭

#### 应用场景

1. 通信
2. 保护用户信息
3. 固件加密
4. 其他，如验证信息完整性

#### 通俗人话解释

1. 对称加密算法，可以理解为家里人说方言，家里人都能听懂而外地的听不懂
2. 非对称加密算法，可以理解为发邮件给他人，要知道他的邮箱是什么（这是公钥），而他进入他自己的邮箱需要密码（密钥）进入邮箱以此查看信息
3. 哈希算法，可以理解为每一段信息都有自己的姓名且都不同名

#### 核心逻辑/原理

1. ==对称加密算法使用相同的密钥进行加密和解密==。发送方和接收方必须共享这个密钥。例如，当用户 A 要发送一条消息给用户 B 时，他们需要事先协商好一个密钥。用户 A 使用这个密钥对消息进行加密，然后将密文发送给用户 B。用户 B 收到密文后，使用相同的密钥进行解密，得到原始消息。
2. ==非对称加密算法使用一对密钥，即公钥和私钥==。公钥可以公开，任何人都可以获取。私钥则由所有者保密。当用户 A 要发送消息给用户 B 时，用户 A 使用用户 B 公布的公钥对消息进行加密，然后发送给用户 B。用户 B 收到密文后，使用自己的私钥进行解密。反过来，用户 B 也可以使用自己的私钥对消息签名，用户 A 可以使用用户 B 的公钥验证签名的真实性。
3. ==哈希算法不是用于加密解密消息，而是将任意长度的数据转换为固定长度的哈希值（也称为消息摘要）==。例如，一个文件或者一段消息可以通过哈希算法生成一个固定长度的哈希值。哈希算法是单向的，即从哈希值很难反推出原始数据。而且，对于不同的输入数据，理想情况下应该生成不同的哈希值。

#### 关键公式/结论

1. 对称加密算法常用的有 DES 和 AES
	1. DES（数据加密标准）：它是一种比较早期的对称加密算法。==DES 采用 64 位的分组长度，但是其中有效密钥长度为 56 位==。由于密钥长度相对较短，现在已经不太安全，容易被暴力破解
	2. AES（高级加密标准）：这是目前应用非常广泛的对称加密算法。==AES 支持 128 位、192 位和 256 位的密钥长度==。它的分组长度为 128 位。安全性很高，使用也较为广泛
2. 对称加密算法的优缺点
	1. 优点就是==加密和解密速度快，加密算法实现简单==，适合用于大量数据的加密
	2. 缺点就是密钥管理困难，由于需要接受和发送双方统一，==密钥在网络环境下一旦泄露，保密系统形同虚设==
3. 非对称算法常用的有 RSA 和 ECC
	1. RSA（Rivest-Shamir-Adleman 算法）：这是目前应用非常广泛的非对称加密算法。==RSA 支持 1024 位、2048 位、3072 位和 4096 位等不同的密钥长度==（目前普遍推荐 2048 位及以上）。它的单次加密数据（分组）长度受限于所选用的密钥长度。安全性很高（基于大整数分解难题），使用也较为广泛。
	2. ECC（椭圆曲线密码学）：这是目前应用非常广泛的非对称加密算法。==ECC 通常支持 256 位、384 位和 521 位等较短的密钥长度（它能用比 RSA 短得多的密钥达到同等安全级别）==。它的核心数学基础为椭圆曲线离散对数问题（ECDLP）。安全性很高，且由于计算速度快、占用资源少，使用也较为广泛。
4. 非对称算法的优缺点
	1. ==密钥管理方便，公钥可以公开分发==，不需要像对称加密那样担心密钥的安全分发问题。并且可以用于数字签名，确保消息的来源和完整性。
	2. ==加密和解密速度相对较慢==，尤其是对于较长的消息或大量数据，计算成本较高
5. 哈希算法常用的有 MD 5 和 SHA
	1. MD5（消息摘要算法第 5 版）：==它可以将任意长度的数据转换为 128 位的哈希值==。但是 MD5 存在安全漏洞，如碰撞问题（不同的数据可能生成相同的哈希值），现在已经不适合用于安全要求较高的场合。例如，早期用于验证文件完整性，但现在已经被更安全的算法取代
	2. SHA（安全哈希算法）：包括 SHA - 1、SHA - 2（SHA - 224、SHA - 256、SHA - 384、SHA - 512）和 SHA - 3。SHA - 1 也逐渐被发现存在安全问题，而 SHA - 2 和 SHA - 3 是目前比较安全的哈希算法。例如，==在数字签名、软件下载验证等场景中，使用 SHA - 256 等算法来验证文件的完整性和真实性==

#### 核心逻辑/原理

明文需要经过四个步骤加密为密文，当然密文也需要这四个步骤的逆向来解密；第一个步骤为密钥扩展，第二个步骤为初始变换，第三个步骤为九次轮操作，第四个步骤为最终轮操作

#### 关键公式/结论

 1. 密钥扩展是将初始密钥矩阵根据公式来进行扩展（这里的运算都是线代中的矩阵运算）
	 1. $W[ i ]$ = $W[i-4]  \oplus W[i-1]$, i 不为 4 的倍数
	 2. $W[ i ]$ = $W[i-4] \oplus T (W[i-1])$, i 为 4 的倍数
		 1. $T (W[i-1])$ 由字循环，字节代换和轮常量异或三个步骤得到
			 1. 字循环，将一列四个字节进行左移，移除的列头回到列尾（i-1 表示的是将 i-1 列进行字循环）
			 2. 字节代换，即根据 S-BOX 查找到相应字节进行更换
			 3. 根据给定的轮常量，与上两步得出的结果进行异或
 2. 初始代换是将明文矩阵与初始密钥矩阵进行异或操作
 3. 九次轮操作也叫主要轮操作由字节代换、行移位、列混合、轮密钥加组成
	 1. 字节代换，根据 S-BOX 查询对应字节如 0 xab, a 为行号，b 为列号，对应的表中数据替换 0 xab
	 2. 行移位，第一行不变，后续的第 n 行左移（n-1）个字节，行头回到行尾
	 3. 列混合，将输入矩阵与给定矩阵相乘
	 4. 轮密钥加，将输入矩阵的每一列与子密钥矩阵的对应的每一列异或（子密钥矩阵由初始密钥矩阵扩展而来）
 4. 追踪轮操作由字节代换、行移位、轮密混加组成，最后输出的矩阵为密文矩阵

#### 核心逻辑/原理

根据选定的一对质数来形成一对密钥

#### 关键公式/结论

1. RSA 数字签名的工作流程：
	1. 首先生成一对密钥——公钥和私钥。公钥用于加密和解签，私钥用于解密和签名
	2. 创建数字签名：发送者（A）使用哈希算法（如 SHA-256）对要发送的数据（M）进行哈希运算，生成一个哈希值（H）。​然后，发送者（A）==使用自己的私钥（d）对哈希值（H）进行加密，生成数字签名（S）==。加密过程可以表示为：S = H^d mod n，其中 n 是公钥和私钥生成过程中使用的两个大素数的乘积。(如果存在压缩，则是对压缩后的数据进行签名)
	3. 发送数据和签名：​发送者将原始数据（M）和数字签名（S）一起发送给接收者（B）
	4. 验证数字签名：​接收者（B）收到数据（M）和数字签名（S）后，首先使用相同的哈希算法对数据（M）进行哈希运算，生成一个哈希值（H'）。​然后，==接收者（B）使用发送者（A）的公钥（e）对数字签名（S）进行解密，得到哈希值（H''）==。解密过程可以表示为：H'' = S^e mod n
	5. 最后，==接收者比较 H' 和 H''。如果两者相等，说明数据在传输过程中没有被篡改==，且确实来自声称的发送者

## 4. Q&A

### Q 1：AES 加密的信息长度是否有限制？是多长？

 A 1: AES 加密的信息长度一般为 128,192 或者 256 个字节

### Q 2：AES 每次加密的信息最短不能短于多少？​

A 2: 最短不能低于 128 个字节

### Q 3: 你能想象到 AES 加密的应用有哪些？​

A 3: 用户信息、固件加密、通信加密

### Q 4：什么是非对称加密 ？和对称加密的区别是什么？​

A 4: 使用一对密钥分为公钥和私钥，公钥用于发送和加密数据，私钥用于解密和数字签名，对称加密发送和接收共用一个密钥

### Q 5：非对称加解密和对称加密谁比较快？​

A 5: 对称加密更快，其加密算法简单主要依靠简单的数学逻辑且硬件，计算量小; 非对称加密涉及大量的指数运算，极其消耗 CPU 资源

### Q 6：非对称加密里面的私钥能不能加密？公钥能不能用来解密？如果可以，那这种应用是什么？​

A 6: 可以，私钥加密，公钥解密的应用为数字签名对发送的数据的哈希值进行加密，这个加密的哈希值也就是“数字签名”

### Q 7：为什么公钥私钥一定“成对”存在？​

A 7: 公钥和私钥在数学上是对称的，由与 d 与 e 是同时出现，根据一系列计算从而得出了公钥和私钥

### Q 8：散列值 (hash 值，MD5 值）除了验证文件完整性，在加密方向还有什么其他应用吗？【简单了解】​

A 8: 可以用于数字签名，来告诉接收方信息的来源

### Q 9：椭圆曲线加密算法（ECC）相较于我们常规的非对称加密算法（RSA）有什么优势和区别？【简单了解】

A 9: ECC 使用少量数据加密即可实现 RSA 相同等级的加密等级，可以节约内存空间大幅度减少 CPU 运算时间

密钥生成的速度也同样极快
