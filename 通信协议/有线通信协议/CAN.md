### 核心逻辑/原理

#### 第一步

每个控制器（MCU）通过 RX 和 TX 与 CAN 收发器连接，收发器通过 CAN_H 和 CAN_L 与总线的 CAN_H 和 CAN_L 相连

![[file-20260615181619901.png]]

这是高速/低速 CAN 总线与主机的连接方式

在闭环网络中两个电阻的作用是防止回波反射（注意这是新概念）和在空闲状态下将 CAN_H 和 CAN_L 电压水平拉高到一致

在开环网络中两个电阻？

#### 第二步

CAN 总线采用差分信号，即两线电压差（VcANH-VcAN⊥L）传输数据位

- 高速 CAN 规定：电压差为 0V 时表示逻辑 1（隐性电平）电压差为 2V 时表示逻辑 0（显性电平）
- 低速 CAN 规定：电压差为 -1.5V 时表示逻辑 1（隐性电平）电压差为 3V 时表示逻辑 0（显性电平）
![[file-20260615183015779.png]]
电平差分显示

#### 第三步

TXD 输出 1，收发器无反应默认输出为 1；当输出 0 时，会让 driver 驱动两个 MOS 管导通，形成 vcc 数值 的电压差

![[file-20260615193758371.png]]

CAN 收发器 TJA 1050

####  第四步

数据帧的格式，标准格式和扩展格式中灰色表示需要为显性电平 0，紫色为隐性电平 1，图形中间的数字表示位数，发送数据前要处于空闲状态即隐性电平 1

![[file-20260616175123186.png|center]]

在时序上方，在不同时序范围内命名了不同的段，SOF 帧起始，RTR 用于区分数据帧还是遥控帧（数据帧电平为 0，遥控帧为 1），SRR 没用但是要给电平 1

IDE 用于区分标准格式还是扩展格式（标准格式为 0，扩展格式为 1），r 0 是保留位

DLC 表示数据段的字节长度，数据段根据 dlc 的指定来发送指定的字节（最大为 8 个字节）

ACK 释放系统总线显示电平为一，而接收方会拉低总线使其显示电平 0

EOF 为帧结束

### 关键公式/结论

1. CAN 总线使用差分信号通信，抗干扰能力强
2. CAN 通信协议无时钟信号为异步通信，且为半双工，多设备发送数据由仲裁判定先后顺序
3. 11 位/29 位报文 ID，用于区别消息功能和决定优先级
4. 数据发送有广播式和请求式（先请求后发送）
5. 遥控帧无数据段其他格式与数据帧一致 ![[file-20260616181932667.png]]
6. 错误帧 file-20260616193552697.png
7. 过载帧 file-20260616193724756.png
8. 位填充：发送发发送五个相同的电平后，自动追加一个相反电平的填充位，接收方会自动移除填充位；其作用是防止波形长时间无变化，将错误帧和过载帧在正常数据流中区分开来，保持 can 总线的活跃状态，防止被认为是总线空闲
