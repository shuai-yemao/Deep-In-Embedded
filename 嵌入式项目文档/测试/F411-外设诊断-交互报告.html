诊断详情

I2C1 BUSY 锁死 — 根因分析与恢复

根因：从设备 BME280 在传输中被复位，主设备未收到 STOP 信号，SR2 BUSY 位持续置 1。

方案：GPIO 模拟 9 个时钟脉冲释放从设备，然后硬复位 I2C 外设。

步骤操作状态
1检查 SCL/SDA 电平，确认总线空闲完成
2GPIO 模拟 9 个时钟脉冲完成
3读 I2C1->SR2 & I2C_SR2_BUSY 确认状态完成
4__HAL_RCC_I2C1_FORCE_RESET() + 延退 10μs完成
5__HAL_RCC_I2C1_RELEASE_RESET()完成
6重新初始化 I2C 时序参数完成
7扫描总线验证：0x76 (BME280) 返回 ACK完成
SPI1 BSY 标志卡死 — 临时恢复代码

根因：连续写入 >256B 未分页，SPI BSY 标志在跨页边界假死。

代码：

spi_busy_recovery.c
// SPI BSY 恢复：软件清除 + 时钟重同步
void SPI_BSY_Recovery(SPI_TypeDef *SPIx) {
    // 1. 尝试读 SR 清除残留 BSY
    uint32_t sr = SPIx->SR;
    (void)SPIx->DR;  // 读 DR 清空 RXNE

    // 2. 如果 BSY 仍置位，CR1 SPE 复位再使能
    SPIx->CR1 &= ~SPI_CR1_SPE;
    volatile uint32_t delay = 10;
    while (delay--);
    SPIx->CR1 |= SPI_CR1_SPE;

    // 3. 发送虚拟字节重新同步
    SPIx->DR = 0xFF;
    while (SPIx->SR & SPI_SR_BSY);
    (void)SPIx->DR;
}
