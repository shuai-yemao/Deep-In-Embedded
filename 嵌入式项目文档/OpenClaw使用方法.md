## 2. 逐点精讲

### 知识点 1：OpenClaw 的基本概念

#### 实际意义

OpenClaw 是 **CherryStudio 的 MCP（Model Context Protocol）工具集**，为 AI 代理提供与外部世界交互的能力。它像 AI 的 " 手脚 "，让 AI 不仅能思考，还能执行实际操作。

####  应用场景

1. **定时任务**：每天早上推送复习、定期检查系统状态
2. **消息通知**：任务完成时主动通知用户
3. **频道管理**：管理微信、Telegram 等通信频道的连接

#### 常见误区

1. **OpenClaw 是一个独立软件**：错误！它是 CherryStudio 内置的 MCP 工具，不需要单独安装
2. **OpenClaw 和 MCP 服务器是一回事**：OpenClaw 只是 MCP 工具集之一，还有其他 MCP 服务器（如 Exa、Browser 等）
3. **只能定时发送消息**：不止！还能管理定时任务、配置频道、监控状态

#### 通俗人话解释

把 AI 想象成大脑，OpenClaw 就是大脑的 " 手脚 "：

- 大脑说 " 我想提醒用户学习 "，手脚就去发通知
- 大脑说 " 每天早上 9 点干活 "，手脚就设置闹钟
- 大脑说 " 帮我连上微信 "，手脚就去扫码连接

#### 核心逻辑/原理

OpenClaw 通过 **MCP 协议** 与 CherryStudio 通信，提供三个核心功能：

1. `mcp__claw__cron` - 定时任务调度
2. `mcp__claw__notify` - 消息通知推送
3. `mcp__claw__config` - 代理配置管理

#### 关键公式/结论

1. OpenClaw = CherryStudio 的 MCP 工具集
2. 提供三大功能：定时任务、通知推送、频道配置
3. 不需要单独安装，开箱即用

### 知识点 2：OpenClaw 的三大核心工具

#### 工具概览

| 工具 | 功能 | 使用场景 |
|------|------|---------|
| `mcp__claw__cron` | 定时任务 | 复习提醒、定期报告 |
| `mcp__claw__notify` | 消息通知 | 任务结果推送、状态更新 |
| `mcp__claw__config` | 配置管理 | 添加微信、检查状态 |

#### 通俗人话解释

- **cron** = 闹钟 + 日历（什么时间做什么事）
- **notify** = 短信通知（有结果了告诉你）
- **config** = 设置中心（连接哪个频道、什么名字）

### 知识点 3：定时任务管理（cron）

#### 实际意义

让 AI 在特定时间自动执行任务，不需要你手动触发。就像雇了一个闹钟管家。

####  应用场景

1. **每日学习提醒**：每天早上 9 点推送复习卡片
2. **定期检查**：每 2 小时检查一次系统状态
3. **一次性任务**：明天下午 3 点提醒我开会

#### 核心逻辑/原理

支持两种调度方式：

1. **Cron 表达式**：灵活的周期调度

   ```
   0 9 * * *    # 每天 9:00
   0 */2 * * *  # 每 2 小时
   0 9 * * 1-5  # 工作日 9:00
   ```

2. **Every 表达式**：简单的间隔调度

   ```
   30m    # 每 30 分钟
   2h     # 每 2 小时
   24h    # 每 24 小时
   ```

#### 关键公式/结论

**Cron 表达式格式**：`分 时 日 月 周`

- `*` = 任意值
- `0 9 * * *` = 每天 9:00
- `0 9 * * 1-5` = 工作日 9:00

**创建任务示例**：

```
mcp__claw__cron
  action: add
  name: "每日学习复习"
  cron: "0 9 * * *"
  message: "推送今日复习卡片"
```

### 知识点 4：消息通知推送（notify）

#### 实际意义

让 AI 主动给你发消息，而不是等你问它。任务完成、状态变化时自动通知。

####  应用场景

1. **复习推送**：定时任务到期时推送复习卡片
2. **任务完成通知**：长时间任务完成后通知你
3. **紧急提醒**：重要事件发生时提醒你

#### 核心逻辑/原理

通过你已连接的频道（微信、Telegram 等）发送消息：

- 默认发送到所有已连接频道
- 可以指定特定 channel_id 发送
- 消息内容支持 Markdown 格式

#### 关键公式/结论

**发送通知示例**：

```
mcp__claw__notify
  message: "复习时间到！今日待复习 3 个知识点"
  channel_id: "微信频道ID"（可选）
```

### 知识点 5：频道配置管理（config）

#### 实际意义

管理 AI 与你通信的渠道，就像手机管理 SIM 卡一样。

####  应用场景

1. **添加微信**：扫码连接微信频道
2. **检查状态**：查看哪些频道已连接
3. **管理设置**：修改代理名称、启用/禁用频道

#### 核心逻辑/原理

支持的操作：

- `status` - 查看当前配置和频道状态
- `add_channel` - 添加新频道
- `update_channel` - 更新频道设置
- `remove_channel` - 删除频道
- `reconnect_channel` - 重新连接（二维码过期时）

#### 关键公式/结论

**添加微信示例**：

```
mcp__claw__config
  action: add_channel
  type: wechat
  name: "微信"
```

## 4. Q&A

### Q 1：OpenClaw 和普通 MCP 服务器有什么区别？

**A 1**：OpenClaw 是 CherryStudio 专门开发的 MCP 工具集，专注于**代理自主能力**（定时任务、通知、频道管理）。普通 MCP 服务器通常只提供特定功能（如文件操作、数据库查询）。OpenClaw 让 AI 更像 " 主动助手 " 而不是 " 被动工具 "。

### Q 2：Cron 表达式怎么理解？

**A 2**：Cron 表达式就像 " 时间密码 "，5 个数字分别代表：

- 第 1 个：分钟（0-59）
- 第 2 个：小时（0-23）
- 第 3 个：日期（1-31）
- 第 4 个：月份（1-12）
- 第 5 个：星期（0-6，0 是周日）

例如：

- `0 9 * * *` = 每天 9 点 0 分
- `0 9 * * 1` = 每周一 9 点
- `*/30 * * * *` = 每 30 分钟

### Q 3：通知可以发到多个频道吗？

**A 3**：可以！默认情况下 notify 会发送到所有已连接的频道。你也可以指定 `channel_id` 只发给某个特定频道。比如只发微信不发 Telegram。

### Q 4：定时任务出错了怎么办？

**A 4**：OpenClaw 会自动重试，并在失败时通知你。你可以通过 `mcp__claw__cron` 的 `list` 动作查看所有任务状态，用 `remove` 动作删除问题任务，重新创建。

### Q 5：OpenClaw 可以和其他 MCP 工具配合使用吗？

**A 5**：当然！OpenClaw 的三大功能（定时、通知、配置）可以和其他 MCP 工具完美配合。比如：

- `cron` + `web_search` = 定时搜索特定信息
- `cron` + `browser` = 定时截图检查网页
- `notify` + `obsidian` = 学习完成后推送笔记链接

## 5. 实际使用示例

### 你刚才搭建的学习流中 OpenClaw 的使用

1. **微信连接**：使用 `config add_channel` 扫码连接微信
2. **复习提醒**：使用 `cron add` 创建每天 9:00 的复习任务
3. **学习通知**：使用 `notify` 发送学习完成通知

### 其他实用场景

**每日学习报告**：

```
cron add
  name: "每日学习报告"
  cron: "0 21 * * *"  # 每晚 9 点
  message: "生成今日学习总结"
```

**项目进度检查**：

```
cron add
  name: "项目进度检查"
  every: "24h"
  message: "检查当前项目进度并报告"
```
